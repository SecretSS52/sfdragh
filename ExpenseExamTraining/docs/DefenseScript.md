# Шпаргалка для защиты

Можно говорить примерно так.

## Короткое объяснение проекта

Это учебный проект учета расходов. Он создан не ради сложной предметной области, а чтобы показать правильную структуру C#-решения: DLL, консольное приложение, тесты, работа с файлами и Git.

## Что показать первым

Открой solution `ExpenseExamTraining.sln`.

Скажи:

> Решение состоит из трех проектов: `ExpenseTracker.Core`, `ExpenseTracker.App` и `ExpenseTracker.Tests`.

## DLL

Открой `src/ExpenseTracker.Core`.

Скажи:

> Это библиотека классов, то есть DLL. В ней находится основная логика проекта: модель расхода, расчет итогов, чтение и запись файлов, отладочные классы.

## Консоль

Открой `src/ExpenseTracker.App/Program.cs`.

Скажи:

> Консольный проект подключает DLL через `ProjectReference`. Он не содержит основной логики, а только вызывает готовые классы из библиотеки.

## Файлы

Открой папку `FileStorage`.

Скажи:

> Для каждого формата есть отдельный класс: CSV, JSON и XLSX. Все они реализуют общий интерфейс `IExpenseStorage`, поэтому консоль может работать с ними одинаково.

## Тесты

Открой `tests/ExpenseTracker.Tests`.

Скажи:

> Тесты написаны на MSTest. Они проверяют расчеты, файловые модули и отладочный класс. Файловые тесты используют подход round-trip: сохранить данные, прочитать обратно и сравнить.

## Отладка

Открой `ExpenseDebugger.cs`.

Скажи:

> Отладочный класс выводит данные только в Debug-сборке через `#if DEBUG`. Это помогает видеть внутренние данные при разработке, но не засоряет Release-сборку.

## Git

Покажи команды:

```powershell
git status
git log --oneline --decorate --graph --all
git branch
git remote -v
```

Скажи:

> В проекте есть коммиты, отдельная ветка и удаленный репозиторий `origin`.
